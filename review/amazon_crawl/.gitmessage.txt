# 커밋 메시지 작성 가이드:
#
# type (필수): commit의 유형을 작성합니다.
#   - feat: 새로운 기능 추가
#   - fix: 버그 수정
#   - docs: 문서 변경
#   - style: 코드 포맷팅, 세미콜론 누락 등 (코드 변경이 없는 경우)
#   - refactor: 코드 리팩토링 (기능 변경 없음)
#   - test: 테스트 추가/수정
#   - chore: 빌드, 패키지 매니저 설정 등 변경 사항 (코드 변경 없음)
#
# scope (선택): 변경된 기능이나 모듈을 명확히 지정합니다.
#
# subject (필수): 50자 이내의 짧은 요약으로, 소문자로 시작하고 명령형으로 작성합니다.
#
# body (선택): 변경한 이유와 상세 설명을 작성합니다.
#
# footer (선택): Breaking Change나 이슈 번호 등을 작성합니다.
#
# 예시:
# feat(auth): OAuth 2.0 로그인 기능 추가
#
